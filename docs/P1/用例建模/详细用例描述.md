## 用例 1：发布快递代取需求

### 用例信息

| 项目 | 内容 |
|------|------|
| 用例编号 | UC-004 |
| 用例名称 | 发布快递代取需求 |
| 参与者 | 发布者（普通学生） |
| 前置条件 | 用户已登录并完成实名认证 |
| 后置条件 | 需求发布成功，进入待接单状态 |

### 基本流程

1. 用户进入"快递代取"模块
2. 系统显示发布需求表单
3. 用户填写快递信息：
   - 快递公司
   - 取件码
   - 快递点位置
   - 期望送达地址
   - 期望送达时间
   - 酬劳金额
4. 用户提交需求
5. 系统验证信息完整性
6. 系统保存需求并发布
7. 系统提示"发布成功"

### 替代流程

**A1：信息不完整（步骤 5）**
1. 系统检测到必填项为空
2. 系统提示"请完善以下信息：xxx"
3. 返回步骤 3

**A2：酬劳金额不合理（步骤 5）**
1. 系统检测到酬劳金额低于最低标准（1 元）
2. 系统提示"酬劳金额不能低于 1 元"
3. 返回步骤 3

### 异常流程

**E1：网络异常（步骤 6）**
1. 系统检测到网络连接失败
2. 系统提示"网络异常，请稍后重试"
3. 用例结束

## 用例 2：接单

### 用例信息

| 项目 | 内容 |
|------|------|
| 用例编号 | UC-005 |
| 用例名称 | 接单 |
| 参与者 | 接单者（普通学生） |
| 前置条件 | 用户已登录并完成实名认证 |
| 后置条件 | 订单状态变为"已接单"，发布者收到通知 |

### 基本流程

1. 用户进入"可接订单"列表
2. 系统显示当前可接的代取订单
3. 用户浏览订单详情
4. 用户点击"接单"按钮
5. 系统确认用户信用分足够
6. 系统更新订单状态为"已接单"
7. 系统通知发布者
8. 系统提示"接单成功"

### 替代流程

**A1：信用分不足（步骤 5）**
1. 系统检测到用户信用分低于 60 分
2. 系统提示"您的信用分不足，无法接单"
3. 用例结束

**A2：订单已被抢（步骤 5）**
1. 系统检测到订单已被其他用户接单
2. 系统提示"来晚一步，该订单已被接单"
3. 返回步骤 2

## 用例 3：完成交易并评价

### 用例信息

| 项目 | 内容 |
|------|------|
| 用例编号 | UC-006 |
| 用例名称 | 完成交易并评价 |
| 参与者 | 发布者、接单者 |
| 前置条件 | 订单状态为"已送达" |
| 后置条件 | 交易完成，双方信用分更新 |

### 基本流程

1. 发布者确认收到快递
2. 发布者点击"确认收货"
3. 系统更新订单状态为"已完成"
4. 系统提示双方进行评价
5. 发布者对接单者进行评价（评分 + 文字）
6. 接单者对发布者进行评价
7. 系统计算并更新双方信用分
8. 系统完成交易结算

### 替代流程

**A1：发布者未确认（超过 24 小时）**
1. 系统自动确认收货
2. 继续步骤 3
