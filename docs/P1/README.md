# Phase 1：需求分析

> 周次：第 2-3 周 | 人均工时：12 小时 | 团队总工时：48 小时

---

## 本阶段目标

1. 通过用户调研获取真实需求
2. 编写规范的需求规格说明书
3. 完成用例建模与用户故事编写
4. 体验 AI 辅助需求分析的能力与边界
5. 完成第一份 AI 协作反思日志

---

## 交付物清单

| 序号 | 交付物 | 格式要求 | 文件 |
|------|--------|---------|------|
| 1 | 调研方案 | 含问卷、访谈提纲 | [调研方案.md](./调研方案.md) |
| 2 | 调研原始数据 | 问卷结果 + 访谈笔记 | [调研数据/](./调研数据/) |
| 3 | "AI 需求盲区挑战"对比分析表 | 按模板填写 | [AI需求盲区挑战.md](./AI需求盲区挑战.md) |
| 4 | 需求规格说明书 | IEEE 830 格式，标注需求来源 | [需求规格说明书.md](./需求规格说明书.md) |
| 5 | 用例图 + 3 个详细用例描述 | 图示 + 文档 | [用例建模/](./用例建模/) |
| 6 | 用户故事（≥15 条） | 含验收标准和优先级排序 | [用户故事.md](./用户故事.md) |
| 7 | AI 协作反思日志 #1 | 按模板填写 | [AI协作反思日志-Phase1.md](./AI协作反思日志-Phase1.md) |
| 8 | Prompt 日志 | 本阶段所有与 AI 的交互记录 | [prompts/](./prompts/) |

---

## 验收标准

- [ ] 有真实的用户调研数据（≥ 20 份问卷 + ≥ 8 人次访谈）
- [ ] "AI 需求盲区挑战"对比分析完整，至少识别出 3 处 AI 输出问题
- [ ] 需求规格说明书符合 IEEE 830 格式，每条需求标注了来源
- [ ] 用例图覆盖至少 3 个角色、15 个用例
- [ ] 用户故事 ≥ 15 条，有验收标准和优先级
- [ ] AI 协作反思日志 #1 内容充实，三个问题都有实质性回答
- [ ] Prompt 日志完整
