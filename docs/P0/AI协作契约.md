## 第一条：AI 使用原则

- AI 是工具，工程师是责任主体
- 所有 AI 输出必须经过人工审查后才能合入项目

---

## 第二条：AI 使用范围

**允许使用 AI 的场景：**

- [ ] 代码补全与建议
- [ ] 文档初稿生成
- [ ] 测试用例生成
- [ ] 代码审查辅助
- [ ] 其他：__________

**禁止直接使用 AI 输出（需额外人工审查）的场景：**

- [ ] 核心业务逻辑代码
- [ ] 安全相关代码（鉴权、加密）
- [ ] 数据库操作代码
- [ ] 其他：__________

**完全禁止使用 AI 的场景：**

- [ ] 个人学习作业中要求独立完成的部分
- [ ] 其他：__________

---

## 第三条：过程记录要求

- 所有与 AI 的交互 Prompt 必须保留日志
- Git commit message 必须标注 `[AI-assisted]` 或 `[Human-written]`
- 每阶段填写《AI 协作反思日志》

---

## 第四条：代码合入规则

- AI 生成的代码必须通过至少一名团队成员的 Code Review
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数
