# AI 协作契约

团队名称：404 Not Found
日期：2026.4.6

---

## 第一条：AI 使用原则

- AI 是工具，工程师是责任主体
- 所有 AI 输出必须经过人工审查后才能合入项目

---

## 第二条：AI 使用范围

**允许使用 AI 的场景：**

- [x] 代码补全与建议
- [x] 文档初稿生成
- [x] 测试用例生成
- [x] 代码审查辅助
- [x] 调试辅助和错误诊断
- [x] 学习新技术和查阅 API 文档
- [ ] 其他：__________

**禁止直接使用 AI 输出（需额外人工审查）的场景：**

- [x] 核心业务逻辑代码（如支付、权限校验）
- [x] 安全相关代码（鉴权、加密）
- [x] 数据库 Schema 设计和关键 SQL
- [x] 第三方库选型和技术决策
- [ ] 其他：__________

**完全禁止使用 AI 的场景：**

- [x] 个人学习作业中要求独立完成的部分
- [x] 涉及学术诚信的原创性内容
- [x] 处理真实用户隐私数据
- [ ] 其他：__________

---

## 第三条：过程记录要求

- 所有与 AI 的交互 Prompt 必须保留日志
- Git commit message 必须标注 `[AI-assisted]` 或 `[Human-written]`
- 每阶段填写《AI 协作反思日志》

---

## 第四条：代码合入规则

- AI 生成的代码必须通过至少一名团队成员的 Code Review
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数

---

## 第五条：违约处理

**违约定义：**
- 未经审查直接将 AI 生成代码合入仓库
- 未按要求标注 commit 来源（AI-assisted / Human-written）
- 在禁止使用 AI 的场景中使用 AI

**处理方式：**

| 违约次数 | 处理方式 |
|---------|---------|
| 首次违约 | 团队内部警告，记录备案 |
| 重复违约 | 在团队会议上公开检讨 |
| 严重违约 | 取消该成员本阶段评优资格 |

**严重违约情形：**
- 未审查的 AI 代码导致线上事故
- 违反学术诚信规定
- 泄露用户隐私数据

---

## 第六条：工具使用规范

**AI 编程助手：**
- 主要使用：通义灵码（Lingma）+ WorkBuddy
- 使用场景：代码补全、单元测试生成、调试辅助

**AI 对话工具：**
- 主要使用：Gemini
- 使用场景：技术调研、文档生成、问题诊断

**AI 代码审查：**
- 使用 GitHub Pull Request 进行人工 Review
- 不依赖 AI 自动审查工具

**AI 测试工具：**
- 使用 Vitest 进行单元测试
- AI 辅助生成测试用例，人工审查后使用
